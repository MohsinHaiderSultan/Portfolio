AI LogGuard (FYP)

AI Explanation

Project Overview

Developed a real-time log analysis tool to detect malicious activities using supervised and unsupervised ML techniques. This was my Final Year Project, focusing on practical cybersecurity applications of AI.

The Problem

Manually sifting through millions of system logs for signs of a security breach is inefficient and prone to human error. Malicious actors often hide their tracks within this "noise."

My Solution

I built a tool that ingests logs in real-time. It uses a supervised model (like a Random Forest) trained on known attack patterns (e.g., SQL injection, brute-force attempts) and an unsupervised model (like Isolation Forest) to detect novel anomalies. The system flags suspicious activity and presents it on a simple dashboard (built with Tkinter).

Python Scikit-learn Pandas Matplotlib Tkinter Firebase

Brute-Force Cracker (OpenMP)

AI Explanation

Project Overview

This project explored high-performance computing concepts by parallelizing a brute-force hash-search algorithm using OpenMP. The goal was to evaluate the performance improvements gained from multi-threaded execution.

The Challenge

A single-threaded password cracker is extremely slow, trying one combination at a time. The challenge was to correctly partition the search space (e.g., all 8-character passwords) and distribute the workload across multiple CPU cores without conflicts.

My Solution

I implemented a C++ program that used OpenMP's `parallel for` directive to split the search. Each thread was assigned a different starting range of password combinations to test. I experimented with different hash functions (MD5/SHA) and measured the linear speed-up as more cores were added, demonstrating the power of parallel programming.

C++ OpenMP Parallel Computing Cybersecurity
